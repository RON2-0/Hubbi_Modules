Módulo de Inventario

Este módulo debe ser un inventario robusto, multifacético y modular, tanto para pequeños inventarios como para grandes.

Funcionalidades:
Dos vistas, una como tabla/Excel utilizando tankstack y una como WMS utilizando Three.js
Tipos de productos, servicios o activos, Elementos fijos y no fijos.
Fecha de vencimiento debe poder activarse o desactivar, desactivado por defecto.
En el futuro este módulo deberá ser compatible para trabajar en conjunto con un módulo de facturación, manteniendo y otros.
- Funcionalidad de etiquetación, para poder generar etiquestas customisables para los productos o activos, con codigos de barra, o qr codes e información de los mismos.
- Funcionalidad de inventario físico, para poder hacer inventarios físicos de los productos o activos que estan en el inventario en al base de datos versus el inventario físico real, tanto atraves de un scanner como de un teclado. Y también sobre todo para poder realizarse atraves de la versión de teléfonos moviles.
- Funcionalidades para poder realizar movimientos de entrada y salida de los productos o activos, tanto atraves de un scanner como de un teclado. Y también sobre todo para poder realizarse atraves de la versión de teléfonos moviles.
- Ingreso de compras e ingreso de notas de credito (Quiza esto en un futuro modulo de compras, para de una vez integrar a los proveedores, etc, no sé como tu consideres).
- Cada item debe poder tener ser de una cantidad entera o decimal. Debe tener una categoría de manera obligatoria. También están los grupos y sub-grupos, Unidad de medida, Unidad de compra, unidad de venta (se deben poder agregar multiples unidades de venta), campos según el tipo de negocio o rubro/perfil seleccionado, garantía, fecha de vencimiento, activo, producto, servicio, etc. Si es un producto, si es exento, si es gravado, si es sujeto a retención, etc. Si es un activo, si es fijo, si es no fijo, etc. Si es un producto debe de poder tener precios de venta (se deben poder agregar multiples precios de venta y estos se deben de asocias a una de las unidades de venta). Se debe poder seleccionar si el item se puede vender sin existencia o no. IVA, RENTA, ETC.
Y no sé cómo funcionaría la lógica pero si es un producto a qué factura pertenece, para poder facilitar el precio con el que se compró y el que se vendió y las ganancias o pérdidas, etc.
